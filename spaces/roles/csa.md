# Cloud Solution Architect (CSA) — Knowledge Layer Context

The CSA is the **architecture-focused execution owner**. In the knowledge layer, CSA contributions focus on:

## What You Own

- **Architecture decisions** — Document design choices, constraints, guardrails
- **Execution monitoring** — Track delivery integrity against committed architecture
- **Customer file enrichment** — Add technical context, dependency maps, risk assessments

## Your Playbooks

- [[architecture-handoff]] — Documenting architecture decisions post-proof
- [[milestone-governance]] — Supporting CSAM on delivery tracking (contributing role)

## Knowledge Layer Habits

| Cadence | Action |
|---|---|
| Per architecture decision | Document in meeting note, link to customer file |
| Per delivery checkpoint | Update guardrail compliance status |
| Weekly | Review execution health, flag constraint violations |

## Handoff Pattern

When delivery completes and moves to value realization:
1. Document final architecture state
2. Capture any deviations from original design and why
3. List production guardrails for ongoing monitoring
4. Link success KPIs for CSAM adoption tracking
