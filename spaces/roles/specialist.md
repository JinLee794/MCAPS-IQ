# Specialist (STU) — Knowledge Layer Context

The Specialist is the **pipeline builder and deal driver**. In the knowledge layer, Specialist contributions focus on:

## What You Own

- **Customer files** — Create and maintain files for actively-pursued accounts
- **Opportunity context** — Document qualification rationale, BANT assessment, solution play alignment
- **Meeting notes** — Capture customer discovery sessions, qualification calls, pipeline reviews

## Your Playbooks

- [[pipeline-qualification]] — Qualifying inbound signals
- [[proof-orchestration]] — Coordinating with SE on technical proofs (supporting role)

## Knowledge Layer Habits

| Cadence | Action |
|---|---|
| Per signal | Create/update customer file + log qualification rationale |
| Per meeting | Capture meeting note with action items |
| Weekly | Review pipeline playbook, update stale customer files |

## Handoff Pattern

When transitioning from STU to CSU (Stage 3+):
1. Ensure customer file has complete team section
2. All meeting notes are captured and linked
3. Opportunity context documents the "why" behind the deal
4. Flag any risks or dependencies in the customer file
