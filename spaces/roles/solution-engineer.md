# Solution Engineer (SE) — Knowledge Layer Context

The SE is the **technical proof executor**. In the knowledge layer, SE contributions focus on:

## What You Own

- **Proof artifacts** — Document POC/Pilot/Demo scope, acceptance criteria, results
- **Technical meeting notes** — Capture architecture discussions, environment details, technical decisions
- **Task hygiene** — Ensure CRM tasks reflect actual proof progress

## Your Playbooks

- [[proof-orchestration]] — Managing POC/Pilot/Demo lifecycle
- [[architecture-handoff]] — Supporting CSA handoff (contributing role)

## Knowledge Layer Habits

| Cadence | Action |
|---|---|
| Per proof | Document scope, criteria, timeline in meeting note |
| Per technical session | Capture decisions and environment state |
| At proof conclusion | Create results summary, update customer file |

## Handoff Pattern

When proof concludes and delivery begins:
1. Document all architecture decisions made during proof
2. List environment prerequisites and configurations
3. Capture acceptance criteria results (pass/fail per item)
4. Link artifacts in customer file for CSA pickup
